      <h1>JavaScript: Your Next Steps</h1>
      
      <p>
        Normally in the last week of class, I like to talk about where you can continue your JavaScript education. This quarter, due to external events, I didn't get around to that. So I'd like to take this opportunity, in these last few minutes together, to talk about good resources to check out if you think JavaScript is a language you'd like to continue studying.

      <p>
        Most <span class="redundant">JavaScript</span> programmers will point to a single book when asked about their professional development: <b id="good-parts">JavaScript: The Good Parts</b> by Douglas Crockford. Crockford did as much as anyone else to show the ways that JavaScript could be a serious, featureful programming language if examined with a critical eye. His book is extremely dense and slow to read, even at only 140 pages, and it is often wrong given what we know now about the ways that JavaScript is actually executed, but it is always wrong in interesting and educational ways.

      <p>
        Axel Rauschmeyer recently released an excellent book on the language itself called <b>Speaking JavaScript</b>, available online <a href="http://speakingjs.com/es5/" class="book">for free</a>. If you want a book that will cover more of the browser and a number of JavaScript libraries, I highly recommend David S. Flanagan's <b>JavaScript: The Definitive Guide</b>, both as a text and as a way to kill small vermin.

      <p>
        One of the best ways to continue learning about this language is to read the source code of projects that use it, since it has a strong open-source community around it. Reading code is also itself an important skill, since it's vital when working on a team with other developers who may not write the same way you do. Here are some starting points:
        
        <ol>
          <li> The jQuery library itself <a href="http://james.padolsey.com/jquery/#v=2.0.3&fn=" class="source">is fairly readable</a>.
          <li> Paul Irish on the Chrome team has a great <a href="" class="video">set</a> of <a href="" class="video">presentations</a> on things he learned by reading the jQuery source. 
          <li> <a href="http://backbonejs.org/docs/backbone.html" class="source">Backbone</a> and <a href="http://underscorejs.org/docs/underscore.html" class="source">Underscore</a> both offer "literate" documentation for their source.
          <li class="self-promotion"> I think the source code for <a href="https://github.com/thomaswilburn/Caret" class="source">Caret</a> is reasonably well-documented and clean.
        </ol>

      <p>
        Finally, I would strongly suggest anyone who is working in front-end development and design to check out NodeJS, the server-side JavaScript runtime. A lot of the newest, most interesting tools for web development are written for Node, including <a href="http://lesscss.org" class="tooling">LESS</a>, <a href="http://gruntjs.com" class="tooling">Grunt</a>, <a href="http://bower.io" class="tooling">Bower</a>, <a href="http://yeoman.io" class="tooling">Yeoman</a>, <a href="http://atom.io" class="editor">Atom</a>, <a href="http://brackets.io" class="editor">Brackets</a>, and <a href="http://lighttable.com" class="editor">Light Table</a>.
